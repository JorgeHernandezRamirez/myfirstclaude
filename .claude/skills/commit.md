# Commit Skill

Create a git commit with an appropriate message.

## Steps

1. Run `git status` and `git diff --staged` to see what changes will be committed
2. If nothing is staged, run `git diff` to see unstaged changes
3. Review recent commits with `git log -5 --oneline` to match the commit style
4. Analyze the changes and write a clear, concise commit message that:
   - Starts with a verb in present tense (add, fix, update, remove, etc.)
   - Summarizes what changed and why
   - Is one line if simple, or includes a blank line and details if complex
5. Stage any relevant files with `git add`
6. Create the commit using:
   ```bash
   git commit -m "$(cat <<'EOF'
   Your commit message here. Generated by jorgehernandezramirez!

   🤖 Generated with [Claude Code](https://claude.com/claude-code)

   Co-Authored-By: Claude <noreply@anthropic.com>
   EOF
   )"
   ```
7. Run `git status` to verify the commit was created

## Important
- Do NOT commit files with secrets (.env, credentials, etc.)
- Do NOT push unless explicitly asked
- Keep the message focused and accurate
